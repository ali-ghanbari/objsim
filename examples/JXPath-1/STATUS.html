1. INTRODUCTION

The JXPath Component contains a set of Java classes that provide static utility methods useful in manipulating Java classes that conform to the JavaBeans Specification naming patterns for bean properties in a dynamic fashion using the XPath syntax. It also supports Java collections, objects with Dynamic Properties (like Map, ServletContext etc) and mechanism for adding support for additional object models.

An innovative and unique feature of this component is the unification of reference syntax between DOM and Java.

2. DEPENDENCIES

The JXPath component is dependent upon the following external components for development and use:
  • Java Development Kit (Version 1.3 or later)
  • JAXP (Version 1.1 or later) build time dependency only - no runtime dependency
  • JUnit Testing Framework (Version 3.7 or later) - for unit tests only, not required for deployment
  • JavaCC (Version 2.0 or later) - only needed when grammar changes - no build or runtime dependency
